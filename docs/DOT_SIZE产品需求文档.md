# DotSize 产品需求文档

> 移动端照片尺寸工具 - 产品需求分析、技术架构、开发实现与测试策略

---

## 1. 产品经理视角 - 需求分析

### 1.1 产品定位

**产品名称**: DotSize（点尺寸）

**产品定位**: 一款面向 iOS/Android 移动端的轻量级照片尺寸处理工具，帮助用户通过拍照快速获取不同尺寸的标准照片。

**核心价值主张**: 快捷、专业、无需专业知识即可获得符合各种尺寸要求的照片。

### 1.2 目标用户群体

| 用户类型 | 使用场景 | 痛点 |
|---------|---------|------|
| 求职者 | 简历照、证件照 | 不知道什么尺寸、需跑到照相馆 |
| 学生 | 考试报名、证件照 | 临时需要、找不到合适尺寸 |
| 社交用户 | 社交媒体头像、封面 | 平台尺寸要求多、裁剪麻烦 |
| 行政人员 | 员工档案、业务证件 | 需要批量处理不同尺寸 |

### 1.3 核心功能清单

#### P0 - 核心功能（MVP 必须）

1. **相机拍照**
   - 后置/前置摄像头切换
   - 闪光灯控制
   - 实时预览辅助线（人像构图）

2. **尺寸模板选择**
   - 预设常用尺寸：一寸、二寸、小一寸、大一寸
   - 自定义尺寸输入（宽度 x 高度 mm/inch）
   - 常用社交媒体尺寸：微信头像、朋友圈、微博封面、Instagram 等

3. **智能裁剪**
   - 手动拖拽裁剪框
   - 自动人像识别居中
   - 缩放与移动

4. **图片导出**
   - 支持 JPEG/PNG 格式
   - 质量/分辨率选择
   - 直接保存到相册

#### P1 - 重要功能

5. **历史记录**
   - 最近处理的照片记录
   - 重复使用历史尺寸

6. **照片导入**
   - 从相册选择已有照片
   - 批量处理

7. **尺寸管理**
   - 保存常用尺寸到收藏
   - 尺寸单位切换（mm/cm/inch/px）

#### P2 - 扩展功能

8. **基础美颜**
   - 亮度、对比度调节
   - 简单滤镜

9. **多图导出**
   - 一次拍照生成多种尺寸

10. **云同步**
    - 账号登录
    - 收藏尺寸跨设备同步

## 2. 架构师视角 - 技术架构

### 2.1 技术选型

#### 推荐方案: Flutter

| 维度 | 选择 | 理由 |
|-----|------|------|
| 框架 | Flutter 3.x | 跨平台性能好，Dart 语言开发效率高 |
| 状态管理 | Riverpod | 轻量、测试友好、社区活跃 |
| 图片处理 | image + flutter_image_compress | 原生 Dart 实现，无需平台特定代码 |
| 相机 | camera | 官方插件，兼容性好 |
| 裁剪 | image_cropper | 支持 Android/iOS 原生裁剪 |
| 存储 | path_provider + sqflite | 本地存储方案成熟 |
| 权限 | permission_handler | 统一管理权限请求 |

#### 备选方案: React Native

适用场景：如果团队熟悉 JavaScript/TypeScript 且需要快速 Web 扩展

### 2.5 技术风险与应对

| 风险 | 应对方案 |
|-----|---------|
| 大图处理内存溢出 | 分块处理、流式导出、降低预览分辨率 |
| 不同设备相机兼容性 | 使用官方 camera 插件，做多设备测试 |
| 裁剪后图片失真 | 使用原生裁剪库，提供多分辨率导出 |
| Android 存储权限 | 使用 MediaStore API，兼容 Android 10+ |
| iOS 照片库权限 | 使用 Photos 框架，正确处理权限请求 |

### 2.6 与团队协作要点

- **产品**: 确认所有预设尺寸模板的具体数值；确认导出格式需求
- **开发**: 提供详细的 API 调用示例；确认图片处理性能指标
- **测试**: 提供多设备测试清单；明确性能基准
